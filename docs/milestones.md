# 交付范围与里程碑

## 目标与范围

- 目标：把采集、入库、管理与同步链路打通，形成可重复交付的 MVP。
- 范围边界：不引入新的跨服务架构；客户端只负责采集与上报；服务端提供入库与管理接口。

## Phase 0：规划与基线

- DL-010：范围拆分与里程碑定义
  - 验收口径：MVP 与迭代清单可追踪，成功指标明确且可映射到后续 Phase。
- DL-020：现有实现盘点与迁移策略
  - 验收口径：复用/淘汰清单与回滚策略明确，避免数据迁移风险不透明。

## 成功指标（MVP）

- 服务端在本地可启动，`/api/health` 返回 200。
- 采集任务入队后能产生可追踪的进度事件。
- Chrome 插件完成采集并上报后，服务端可看到任务记录。
- MCP 客户端可完成基础连接与工具调用。

## 依赖与边界

- 本地依赖：Node.js、better-sqlite3、（可选）Playwright。
- 客户端依赖：Chrome 扩展、VSCode 扩展。
- 不在本次范围：多服务拆分、云端部署、复杂权限体系。
